# PMO Project Pack

A concise set of **PMO governance and reporting artifacts** created for a fictional digital product MVP.  
This project pack demonstrates how I structure project oversight, stakeholder communication, risk tracking, and status reporting in a PMO or agile delivery environment.

All documents are provided in **PDF format** for easy review.

---

## Project Overview

**Scenario:**  
A fictional non-profit organization implementing a Scholarship Management Portal to replace a manual, spreadsheet-based application and review process.

**Objective:**  
Deliver a minimum viable product (MVP) that enables:
- Online scholarship applications
- Reviewer evaluation workflows
- Administrative configuration
- Status tracking and notifications

---

## Artifacts Included

### Core PMO Artifacts
- **01 — Project Charter**  
  Defines objectives, scope, success metrics, roles, assumptions, constraints, milestones, and key risks.

- **02 — Business Case (Lite)**  
  Evaluates solution options, benefits, costs, risks, and provides a recommendation.

- **03 — Stakeholder Register & RACI**  
  Maps stakeholder influence and clarifies responsibility across key activities.

- **04 — Communication Plan**  
  Establishes cadence-based communication across delivery teams, stakeholders, and leadership.

- **05 — RAID Log**  
  Tracks risks, assumptions, issues, and decisions in a structured PMO format.

- **06 — Dependency Tracker**  
  Identifies cross-team and delivery dependencies, ownership, and mitigation actions.

- **07 — Integrated Timeline (Gantt)**  
  High-level milestone view across discovery, build, testing, release, and closure.

- **08 — Weekly Status Report (Template)**  
  Standardized reporting format with RAG status, progress, risks, and decisions.

- **09–10 — Weekly Status Reports (Sample Weeks)**  
  Two completed examples showing realistic project reporting progression.

### Governance Extension
- **11 — Change Log**  
  Simple change control log for scope and decision governance.
